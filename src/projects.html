  • Bookstore
    A simple database for bookstore record keeping. The goal of this project was to understand and apply the Command, Memento, and Decorator patterns by creating a persistent database for books. Features:
    • Command Pattern: All actions on the database are serializable objects. After each action is taken, the action is logged so that the actions can be replayed to recover the database (simulates journaling).
    • Memento Pattern: After a fixed number of actions, dump the current database contents to persistent storage and clear the journal of logged actions.
    • Decorator Pattern: Decorate (compose) the original inventory in our memento object. The original inventory is in-memory only. Decorating the simple inventory allows us to add the command and memento functionality mentioned above.

  • Autograder
    Viewing access available upon request
    A test/grading framework built to work with Gradescope. Automates compilation, running, testing, and debugging of students' code. Makes developing test scripts and added features agnostic of the assignment/course. Features:
    • Submission limit: Enforce a maximum number of submissions. Encourages students to write their own unit tests rather than submit hundreds of times until all tests pass.
    • Checksum verification: Identifies duplicate submissions to avoid incorrectly using up submission counts.
    • Automated debugging: Uses core dumps to detect code that crashed. The core dump allows for deterministic debugging and printing out the stack trace at the time of the crash.
